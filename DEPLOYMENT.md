# CRS Test Data Generator — Deployment Guide

## Architecture Overview

```
┌─────────────────────────────────────────────────────┐
│  Final Installer: CRS-Generator-Setup-1.0.0.exe     │
│                                                     │
│  ┌─────────────────┐  ┌──────────────────────────┐  │
│  │  Electron App    │  │  Python Backend (bundled) │  │
│  │  (Vite + React)  │  │  crs_cli.exe             │  │
│  │  dist/           │  │  cbc_cli.exe             │  │
│  │  electron/       │  │  fatca_cli.exe           │  │
│  │                  │  │  error_injector.exe      │  │
│  └─────────────────┘  └──────────────────────────┘  │
└─────────────────────────────────────────────────────┘
```

- **Development mode**: Electron calls system Python via `python -m crs_generator.cli ...`
- **Production mode**: Electron calls bundled `crs_cli.exe ...` (no Python needed on user's machine)
- **Auto-updates**: App checks GitHub Releases for new versions on startup

## Prerequisites

### For Development
- **Node.js 18+** — `node --version`
- **Python 3.10+** — `python --version`
- **Git** — `git --version`

### For Building Releases
- All of the above, plus:
- **PyInstaller** — `pip install pyinstaller`
- All Python deps — `pip install -r requirements.txt`

## Quick Start (Development)

```bash
# 1. Clone and setup
git clone https://github.com/YOUR_USERNAME/crs-testdata-generator.git
cd crs-testdata-generator

# 2. Python setup
python -m venv .venv
.venv\Scripts\activate          # Windows
pip install -r requirements.txt

# 3. Electron setup
cd electron-app
npm install

# 4. Run in dev mode
npm run build                   # Build Vite frontend
npm run electron                # Launch Electron app
```

## Branch Strategy

```
main              ← Tagged releases only (v1.0.0, v1.1.0, ...)
  └── develop     ← Daily work happens here
       ├── feature/my-feature
       ├── fix/bug-name
       └── hotfix/urgent-fix (branch from main for emergencies)
```

### Rules
- **Never push directly to `main`** — only merge from `develop` via PR
- **`develop`** is the default working branch
- **Feature branches** are created from `develop` and merged back via PR

## Release Process

### Option A: Automated (Recommended)

Push a version tag and GitHub Actions builds + publishes the installer automatically.

```bash
# 1. Merge develop into main
git checkout main
git pull origin main
git merge develop

# 2. Bump version (creates git tag automatically)
cd electron-app
npm version patch      # 1.0.0 → 1.0.1 (bug fixes)
# npm version minor    # 1.0.0 → 1.1.0 (new features)
# npm version major    # 1.0.0 → 2.0.0 (breaking changes)
cd ..

# 3. Commit the version bump and push with tag
git add .
git commit -m "release: v1.0.1"
git tag v1.0.1
git push origin main --tags

# 4. GitHub Actions automatically:
#    - Builds Python backend with PyInstaller
#    - Builds Vite frontend
#    - Packages Electron installer
#    - Uploads to GitHub Releases

# 5. Sync version back to develop
git checkout develop
git merge main
git push origin develop
```

### Option B: Manual Local Build

```bash
# 1. Build Python backend (from project root)
python build_python_backend.py
# Creates: electron-app/python-dist/crs_cli.exe, cbc_cli.exe, etc.

# 2. Build Vite frontend
cd electron-app
npm run build
# Creates: electron-app/dist/

# 3. Build Electron installer
npx electron-builder --win --x64
# Creates: electron-app/dist-electron/CRS-Generator-Setup-1.0.0.exe
#          electron-app/dist-electron/latest.yml

# 4. Create GitHub Release manually
#    - Go to: github.com/YOUR_USERNAME/crs-testdata-generator/releases/new
#    - Tag: v1.0.0
#    - Upload: CRS-Generator-Setup-1.0.0.exe + latest.yml
#    - Publish
```

## How Auto-Updates Work

```
User opens app (v1.0.0)
    ↓
App checks: github.com/YOUR_USERNAME/crs-testdata-generator/releases/latest
    ↓
Finds: v1.0.1 available (reads latest.yml)
    ↓
Downloads new installer in background
    ↓
Notifies user: "Update available — restart to install"
    ↓
User clicks restart → app updates to v1.0.1
```

The `latest.yml` file (auto-generated by electron-builder) tells the updater where to find the new `.exe` and its checksum.

## Configuration Checklist

Before your first release, update these values:

### `electron-app/package.json`
```json
"publish": [
  {
    "provider": "github",
    "owner": "YOUR_GITHUB_USERNAME",    ← Change this
    "repo": "crs-testdata-generator"    ← Change this if different
  }
]
```

### GitHub Repository Settings
1. **Create `develop` branch** and set as default
2. **Protect `main` branch**:
   - Require pull request before merging
   - Require 1 approval
   - No force pushes
   - No deletions
3. **Enable GitHub Actions** (should be enabled by default)
4. **No additional secrets needed** — `GITHUB_TOKEN` is automatic

## Troubleshooting

### PyInstaller build fails
```bash
# Clean and rebuild
rmdir /s /q build
rmdir /s /q electron-app\python-dist
python build_python_backend.py
```

### Electron-builder fails
```bash
cd electron-app
rmdir /s /q dist-electron
rmdir /s /q node_modules
npm install
npm run build
npx electron-builder --win --x64
```

### Auto-update not working
- Ensure `latest.yml` is uploaded alongside the `.exe` in the GitHub Release
- Auto-update only works in **installed** apps, not dev mode
- Check the Electron console for `Auto-updater` log messages

### App says "Python not found" in dev mode
- Ensure Python is on your PATH: `python --version`
- Or activate your venv: `.venv\Scripts\activate`
